Tipos de repertorio de instrucciones

En Sicome 2.0 existen dos tipos de repertorio:

  • Repertorio Microprogramado
  • Repertorio Cableado

Formato del Repertorio de Instrucciones realizado mediante Control Microprogramado

El formato del Repertorio Microprogramado es un fichero de texto plano en el que usted debe definir el nombre de cada instrucción, así como su microprograma correspondiente. El formato del fichero es sencillo. No pueden existir en él líneas en blanco, ni se permiten comentarios. En cada línea del fichero se declarará una instrucción, que consta de los siguientes campos separados por espacios:

ADD

true

8000100 1100 20200

neumónico

bandera

Microprograma

El neumónico es el nombre de la instrucción. La bandera indica si la instrucción lleva parámetro o no. Por último, el microprograma está constituido por una secuencia de micropalabras escritas en hexadecimal. Éstas definen la función de la instrucción. La computadora permite hasta un máximo de 32 instrucciones, por lo que si se definen más en un repertorio, éste sólo reconocerá las 32 primeras. Es muy importante no dejar líneas en blanco dentro del fichero, pues esto daría lugar a un error a la hora de interpretar el repertorio.

Para ver con mayor claridad el formato definido aquí se ha colocado el contenido de un fichero de ejemplo:

CRA false 8200

CTA false 10200

ITA false 18200

CRF false 90200

CTF false 98200

SFZ false 100500 200

ROR_F_ACC false 30200

ROL_F_ACC false 28200

ADD true 8000100 1100 20200

ADDI true 8000100 1100 8000100 1100 20200

STA true 8000100 2100 1000200

JMP true 2000200

JMPI true 8000100 1100 2000200

CSR true 8000100 403100 1000100 200200

ISZ true 8000100 1100 4100 1000100 600 200200

·         Cambio del Ciclo de Búsqueda

En SiCoMe v2.0 es posible modificar el ciclo de búsqueda definido previamente. Para ello al comienzo del fichero de repertorio debemos colocar el símbolo $ en la primera línea del fichero. En las tres siguientes líneas debemos colocar la palabra clave CB, seguida de la micropalabra correspondiente a las microoperaciones que se van a realizar. Sólo se permitirán tres ciclos para el ciclo de búsqueda. Al terminar estos tres ciclos, en la línea siguiente se colocará otra vez el símbolo especial $, para indicar a la computadora que lo que sigue a continuación serán las distintas instrucciones que vamos a desarrollar.

Una cosa que hay que tener en cuenta, es que una vez que modificamos el ciclo de búsqueda, para volver al inicial tenemos que realizar los pasos anteriormente descritos, solo que ahora se colocaran las micropalabras que realizan el ciclo de búsqueda convencional, es decir, que el repertorio será a partir de entonces como el que se describe a continuación, no como el del ejemplo anterior. Para no tener que indicar el ciclo de búsqueda en el repertorio deberemos iniciar la computadora desde cero.

Para que esto quede más claro, a continuación se verá un ejemplo en el que modificamos el ciclo de búsqueda para que en el último ciclo se ejecute la microoperación GPR(AD) à MAR:

$

CB 4000100

CB 0101100

CB B000300

$

CRA false 8200

CTA false 10200

…

Formato del Repertorio de Instrucciones realizado mediante Control Cableado

El formato del Repertorio Cableado es un fichero de texto plano en el que usted debe definir el nombre de cada instrucción y una serie de argumentos que corresponden a cada instrucción. En cada línea del fichero se declarará una instrucción, que consta de los siguientes campos separados por espacios:

ADD

true

3

q1

neumónico

bandera

nº ciclos

id instrucción

Al igual que para el Repertorio Microprogramado, el neumónico es el nombre de la instrucción y la bandera indica si la instrucción lleva parámetro o no. El nº ciclos indica cuantos ciclos posee nuestra instrucción que previamente ha sido definida. El id instrucción es el identificador que el usuario le debe dar a cada instrucción, siendo siempre la letra q seguida de un índice, el cual deberá coincidir con el orden en que han sido definidas en el repertorio.

Para una mejor comprensión, a continuación se presenta un ejemplo de Repertorio Cableado:

HALT false 0 q0

CRA false 1 q1

CTA false 1 q2

ITA false 1 q3

CRF false 1 q4

CTF false 1 q5

SFZ false 1 q6

ROR_F_ACC false 1 q7

ROL_F_ACC false 1 q8

ADD true 3 q9

ADDI trae 5 q10

STA true 3 q11

JMP true 1 q12

JMPI true 3 q13

CSR true 4 q14

ISZ ture 5 q15

NOTA: en el Repertorio Cableado no se puede modificar el ciclo de búsqueda, éste vendrá definido por el usuario en el Controlador Cableado, siendo además su inclusión obligatoria.
