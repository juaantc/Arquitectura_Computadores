Problemas relacionados con el servidor

En el momento de establecer la conexión con el servidor, en el de desconectarse de éste o bien, en el de reclamar o recibir algún servicio pueden aparecer errores. Normalmente éstos se reflejan en un mensaje dentro de un cuadro de diálogo. Sus causas pueden ser las más diversas: fallos en la red, información incorrecta o incompleta, etc. Para resolver posibles dudas que puedan surgir sobre estos mensajes se ha creado esta sección de ayuda.

Problemas al conectar con el servidor

1. "No se puede abrir un puerto de E/S con el servidor"

Este error es muy común y suele darse cuando se intenta acceder al servidor sin estar conectado a internet o bien, si se está conectado, existe algún problema en la red. También puede ser que el host que se ha especificado como servidor no sea el correcto, con lo cual no tenga el puerto de llamada abierto. Compruebe los parámetros del servidor en la ventana de preferencias (en caso de duda consulte Parámetros del servidor). Si todo está bien, compruebe que está conectado a internet y, si lo está, que el host servidor está accesible. Para ello bastará con hacer un ping a dicha máquina.

2. "Ya existe una conexión"

Cuando se obtiene este mensaje, indica que usted ya está conectado usando su login y su password en desde otro equipo distinto al actual. En tal caso, debe cerrar la conexión de aquel equipo primero y volver a intentarlo desde el equipo actual. También puede ocurrir haya cerrado el simulador sin haber cerrado antes la conexión de forma correcta. En cuyo caso, el registro de conexiones no reflejará que usted ya no está conectado hasta pasados unos instantes. En este caso, vuelva a intentar conectar pasados unos minutos.

3. "No se aceptan más conexiones"

El servidor de prácticas puede ser configurado de manera que sólo acepte un número determinado de conexiones simultáneas. Su usted recibe este mensaje significará que ese número ha sido cubierto, por lo que deberá esperar y volver a intentarlo más tarde, cuando el número de usuarios conectados haya disminuido.

4. "Clave incorrecta"

Este mensaje se explica por sí solo. Cuando se obtiene significa que bien el login o bien el password o ambos, no son correctos.

5. "Imposible conectar con el host"

El host introducido en la configuración del servidor es un host inexistente o bien por problemas en la red es inaccesible. Compruebe los parámetros de configuración del servidor. En caso de dudas consulte la sección de Parámetros del servidor.

Problemas al acceder al servidor

1. "Error al cargar repertorio"

Este mensaje viene acompañado del error que ha provocado que el fichero de repertorio no se pueda cargar. Se recomienda consultar la sección Problemas cargando ficheros para una explicación más detallada de los posibles errores que pueden aparecer.

2. "Error al cargar programa"

Este mensaje viene acompañado del error que ha provocado que el fichero de repertorio no se pueda cargar. Se recomienda consultar la sección Problemas cargando ficheros para una explicación más detallada de los posibles errores que pueden aparecer.

3. "No puede guardar el archivo. Sobrepasa la cuota"

Este mensaje se obtiene al intentar guardar un fichero desde el editor cuando todos los ficheros que ya existen en su cuenta de usuario más el que está intentando guardar, superan en tamaño la cuota máxima permitida por usuario. Para poder guardar el fichero deberá borrar antes alguno de los que se encuentran en la cuenta de usuario.

4. "Se ha producido un error de comunicación"

Cuando esté guardando un fichero, puede que durante el envío de datos se produzca algún error y la información que estaba mandando llegue al servidor corrompida. En este caso usted recibirá en pantalla este mensaje de error. Bastará con intentarlo de nuevo hasta que el fichero al completo llegue sin errores al servidor.

Problemas al desconectar del servidor

1. "No se encuentra conectado al servidor"

Es posible que alguna vez intente desconectar del servidor cuando ya ha realizado esta operación. En ese caso recibirá este mensaje en pantalla.

2. "Fallo en la desconexión. Sigue conectado al servidor"

Debido a un fallo al intentar la desconexión, ésta no se ha podido consumar, por lo que la conexión sigue abierta. En estos casos se recomienda volver a intentar desconectarse, pues puede que antes ocurriera un error en la comunicación y la petición no fuese reconocida.
